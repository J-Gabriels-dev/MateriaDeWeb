Estruturando Páginas Web com HTML

O HTML (Hypertext Markup Language) é uma das três linguagens fundamentais do desenvolvimento frontend, ao lado do CSS e do JavaScript. Sua função principal é definir a estrutura e a semântica das páginas web, dizendo ao navegador o que cada parte da página representa, sem se preocupar com aparência visual, que é responsabilidade do CSS.

1. Breve História do HTML

O HTML surgiu em 1992, criado por Tim Berners-Lee no CERN. O HTML 4.0, lançado em 1997, trouxe suporte ao CSS. O XHTML 1.0 reformulou a linguagem com regras mais rígidas baseadas no XML. Já o HTML5, lançado em 2014, introduziu novos elementos semânticos e APIs web. Hoje o padrão é mantido como um Living Standard pelo WHATWG em parceria com o W3C, evoluindo continuamente sem número de versão fixo.

2. Elementos e Atributos

Toda a linguagem HTML é construída sobre elementos. Cada elemento é composto por uma tag de abertura, um conteúdo e uma tag de fechamento. Os atributos ficam dentro da tag de abertura e fornecem informações adicionais ao elemento, possuindo sempre um nome e um valor entre aspas separados por sinal de igual.

Estrutura básica de um documento

Todo arquivo HTML começa com !DOCTYPE html, que indica ao navegador qual padrão utilizar. O elemento html envolve todo o documento. Dentro dele, o head contém informações que não aparecem na tela, como o título da aba. Já o body contém tudo que o usuário visualiza na página.

3. Formatação de Texto

O HTML oferece seis níveis de títulos, de h1 (maior) a h6 (menor). Parágrafos são definidos com p. Para destaque visual, usamos b ou strong para negrito e i ou em para itálico. Quebras de linha são inseridas com br, um elemento vazio que não possui tag de fechamento separada.

4. Listas

Existem três tipos de listas em HTML. A lista ordenada com ol numera os itens automaticamente. A lista não ordenada com ul exibe marcadores visuais. A lista de definição com dl agrupa termos e suas descrições. Todas podem ser aninhadas, criando listas dentro de listas com indentação automática pelo navegador.

5. Links e Navegação

Links são criados com o elemento a e o atributo href. Podem ser externos (apontam para outros sites), relativos (páginas do mesmo site) ou internos (âncoras dentro da mesma página usando o símbolo # seguido do id do destino). O texto do link deve sempre ser descritivo e significativo, evitando expressões vagas como "clique aqui".

6. Imagens

Imagens são inseridas com o elemento vazio img. O atributo src define o caminho do arquivo e o atributo alt fornece um texto alternativo, essencial para acessibilidade e mecanismos de busca. Desde o HTML5 é possível adicionar legenda usando figure e figcaption.

7. Tabelas

Tabelas organizam dados em linhas e colunas. O elemento table define a tabela, tr cria linhas, th define cabeçalhos e td contém os dados. Para tabelas longas recomenda-se usar thead, tbody e tfoot. Os atributos colspan e rowspan permitem mesclar células entre colunas e linhas respectivamente.

8. Formulários

Formulários permitem que o usuário envie dados ao servidor. São definidos com form, que recebe os atributos action (URL de destino) e method (método HTTP). O elemento input cria campos conforme o valor do atributo type, como text, password, radio, checkbox e submit. Outros elementos incluem select para listas suspensas, textarea para textos longos e label para rotular campos.

9. Conceitos Finais

Elementos de bloco e inline

Elementos de bloco como h1, p e ul sempre começam em uma nova linha. Elementos inline como a, b e img ficam na mesma linha do conteúdo ao redor.

Agrupadores genéricos

O elemento div agrupa conteúdo como bloco, enquanto o span agrupa conteúdo inline. Ambos são amplamente usados em conjunto com CSS e JavaScript para criar componentes de interface personalizados.

Semântica e HTML5

O HTML5 trouxe elementos semânticos como header, footer, nav, article, section e aside, que tornam o código mais legível e acessível, ajudando tanto desenvolvedores quanto mecanismos de busca a compreenderem melhor a estrutura da página.
